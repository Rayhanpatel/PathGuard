# PathGuard Edge — Hackathon Demo Script (3 Minutes)

> **Objective:** Hit the Ironsite judging rubrics: Real-World Impact (Why edge?), Scientific Rigor (Architecture/Data), and Execution (Live Demo).
> **Tone:** Fast-paced, confident, technical.
> **Setup:** Have the Streamlit app running in the background. Stop your Slack/Discord notifications. Take a deep breath.

---

## 🎬 0:00 - 0:30 | The Hook & The Architecture

**Visual:** The camera is on you, or you are showing the `Architecture Diagram` from your GitHub README.

**Script:**
"Hi, I’m Rayhan, and this is **PathGuard Edge**.

Over 800 workers die each year in construction, primarily from slips, trips, and falls. But you cannot solve this by streaming 4K bodycam footage to a cloud GPU server—construction sites are dynamic, unstructured environments that lack the cell service and bandwidth for cloud streaming.

Spatial safety intelligence *must* live on the edge.

So, we built a hybrid, no-training safety system that combines an always-on geometric classical computer vision layer with heavy, on-device Vision-Language Models. It is designed to run completely offline on an Apple Neural Engine, an Android phone, or an $80 Raspberry Pi 5."

---

## 🎬 0:30 - 1:15 | Part 1: Cactus Narrator (Semantic Understanding)

**Visual:** Switch to the `Cactus Narrator` Streamlit tab.
**Action:** Select the `11_prep_masonry.mp4` video (from their provided dataset). Set Max Tokens to 256. Click **Start**.

**Script:**
"Our pipeline starts with the **Cactus Engine**.

Instead of relying on static, hardcoded object detectors that fail in the real world, we run a highly optimized 1.6-Billion parameter Vision Language Model locally on the device's Neural Engine.

As the worker moves through the site, the VLM reads the scene and generates a massive, 256-token transcript of *everything* it sees. *(Point to the transcript box with your mouse)* Look at this—it correctly identifies rebar, scaffolding, and concrete blocks from the Ironsite masonry dataset.

*(Click the "Summarize Session" button)*
From this transcript, we automatically extract up to 256 dynamic text prompts and build a structured 'RCP Telemetry' JSON report for the site manager."

---

## 🎬 1:15 - 2:00 | Part 2: PathGuard HUD (Spatial Safety)

**Visual:** Switch to the `PathGuard HUD` Streamlit tab.
**Action:** Select the same masonry video. Under the *Dynamic Prompts* dropdown, select the prompt list you just generated. Turn on all the ML models (GroundedDINO, Depth Anything). Click **Start**.

**Script:**
"But semantic understanding isn’t enough—you need to know if a hazard is actually in your physical walking path. We feed those 256 custom prompts into the **PathGuard HUD**.

Notice the yellow trapezoid on screen. We don't care about the whole site; we only care about the worker's forward walking corridor.

Every single frame, a lightweight classical computer vision pipeline measures the physical occupancy of that corridor. Meanwhile, Grounding DINO and Depth Anything V2 run on a scheduled loop to search for the specific objects Cactus found earlier.

*(Point your mouse at the screen when the state machine triggers)*
Watch the state machine evaluate the physical corridor. Because a concrete block enters the walking lane, it immediately escalates from 'Clear' to 'Obstacle Ahead', and then to 'Trip Risk', triggering a localized audio alarm."

---

## 🎬 2:00 - 2:30 | The Climax (Product Viability / Data Engineering)

**Visual:** Wait for the video to finish. The green **"Download Enriched Telemetry (JSON)"** button will pop up.
**Action:** Click the button to download the file. Open the JSON on screen so the judges can see the code.

**Script:**
"Finally, a safety tool is useless if the site manager doesn't get the data.

When the shift is over, PathGuard bridges the gap between semantic understanding and spatial physics. *(Show the JSON file on screen)*.

It takes the site report generated by Cactus and injects the actual physical hazard data—peak corridor occupancy, depth buckets, and exact trip risk event timestamps.

This isn't an AI sandbox; it is a scientifically rigorous, closed-loop telemetry pipeline designed for real-world hardware.

Thank you."
